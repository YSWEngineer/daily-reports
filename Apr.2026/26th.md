## 💡 本日の学び・気付き
###  nginxとは？
- **nginx（エンジンエックス）は Webサーバー**：ブラウザからのリクエストを受けて、HTMLファイルを返す役割を担う
- **nginx は静的コンテンツの配信が得意**：HTML や CSS、画像などをそのまま返すのが速い。逆に動的なコンテンツは nginx 単体では提供できない
- **Webアプリケーションサーバーと連携**：nginx は単体では動的コンテンツを生成しないが、Webアプリケーションサーバーにリクエストを転送することで、結果として動的コンテンツを提供できる

###  nginxのセットアップ
#### SSH接続とSSH接続の切り替え
##### SSH接続とは何か？
簡単に言うと、「**自分のパソコン（Mac）から、遠くにあるサーバーを操作する**」方法。
```shell
VNCコンソール = サーバーの画面を遠隔でそのまま操作する仕組み
SSH接続 = ネットワーク越しに安全にサーバーへログインして操作する方法
```
どちらも「**サーバーを操作する**」という目的は同じだが、**接続方法が違う**だけ。
##### VNCコンソール
**イメージ：サーバーに直接繋がったモニター**
```shell
[Yoshiwo] → [ブラウザ] → [さくらのVPS] → [サーバーの画面]
                              ↑
                        ここを経由している
```
###### 特徴
- ✅ ブラウザから接続できる
- ✅ サーバーが起動しなくても使える（緊急時に便利）
- ✅ 設定不要ですぐ使える
- ❌ コピペがしにくい
- ❌ 画面が小さい・見にくいことがある
###### 使う場面
- 初めてサーバーを設定するとき
- サーバーが起動しないなどの緊急時
- SSH接続が使えないとき

##### SSH接続
**イメージ：自分のパソコンから直接サーバーに接続**
```shell
[Yoshiwoのターミナル] ←→ [サーバー]
      ↑
    直接繋がる
```
###### 特徴
- ✅ コピペが自由自在
- ✅ 動作が軽快
- ✅ 複数のタブで作業できる
- ✅ 実務で使われる標準的な方法
- ❌ 初回だけ接続設定が必要
- ❌ ネットワークが切れると接続も切れる
###### 使う場面
- 普段の開発作業（これが標準）
- ファイル編集やコマンド実行
- ログ確認
- 長時間作業するとき

##### SSH接続に切り替えるとは？
**今の状態**
```shell
[Mac] → [ブラウザ] → [VNCコンソール] → [サーバー]
```
- VNCコンソールの画面でサーバーを操作している
- コピペができなくて不便

**SSH接続に切り替えた後**
```shell
[Macのターミナル] ← 直接SSH接続 → [サーバー]
※ VNCコンソールは閉じてもOK（使わない）
```
- Mac のターミナルから直接サーバーを操作
- コピペが自由にできる

#### SSH接続の手順（step-by-step）
##### ステップ1：サーバーのIPアドレスを確認
まず、さくらの VPSコントロールパネルでサーバーの IPアドレスを確認する。
###### 手順
1. さくらのVPSにアクセス
2. ID と パスワードを入力してログイン
3. 契約中のサーバー一覧が表示
4. 基本情報またはネットワークから該当サーバーの IPアドレスを確認
  - `160.XXX.XXX.XXX`のような数字の並びが IPアドレス

##### ステップ2：Macのターミナルを開く
いつものようにターミナルを開けばOK。

##### ステップ3：SSH接続コマンドを実行
ターミナルに以下のコマンドを入力する。
```shell
% ssh yoshiwo@IPアドレス
```
だが、私の場合は最初の設定でポート番号を変更したので、ポート番号を指定する内容もコマンドに含ませる。
```shell
% ssh -p 設定したポート番号 yoshiwo@IPアドレス
```
```shell
Enter passphrase for key '/Users/yoshiwo/.ssh/id_XXXXXXX':
```
が表示されたら、
1. 慎重に PassPhrase（パスフレーズ）を入力
2. 画面には何も表示されないがマシンの中ではしっかり入力されている
3. 正しく入力できたら`return`キー

以下のようなプロンプトが表示されれば SSH接続は成功！

### 復習
#### ログイン済みの画面表示
VNCコンソールで接続できると黒い画面（ターミナル）が表示。
もし、ログインプロンプトが出ている場合は`login:`にユーザー名を入力して`return` → パスワードを入力して`return`という手続きが必要となる。

私の場合は既にログイン済みの状態になっていて、`yoshiwo@XXX-XXX-XXXXX:~$`と表示されていた。
- `yoshiwo`：ユーザー名
- `XXX-XXX-XXXXX`：サーバー名
- `~`：ホームディレクトリにいることを示す（ターミナルと同じ）
- `$`：コマンド入力待ち

#### Passphraseとは？
Passphrase（パスフレーズ）とは、SSH鍵を使うときに必要なパスワードのこと。
##### 普通のPasswordとの違い
|用語   |何のパスワード？   |いつ使うの？   |
|---|---|---|
|Password   |サーバーへのログインパスワード   |VNCコンソールでログインするとき   |
|Passphrase   |SSH鍵を使うためのパスワード   |SSH鍵認証で接続するとき   |
