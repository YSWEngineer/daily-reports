##### 本日の講師
メンターの町田さん✨

##### Jekyll（ジキル）
- 静的ページを作れるジェネレーター。
- これを使ってブログを書くと、github pages でブログがホストできるため、サーバー代がかからずオリジナルのブログが書ける。
- https://jekyllrb-ja.github.io/docs/

##### Chrome DevTools
- DevTools上で直接スタイルを変更することで、CSSファイルを書かずにデザイン調整ができる。
- 数値の操作
        - カーソルの上下キーで数値を増減できる。
        - `shift`キーを併用すると、`10`ずつ変更できる。

##### ブラウザと CSS
- ブラウザには元々デフォルトのスタイルが存在する。
- デフォルトのスタイルはブラウザ毎に異なる。

##### reset CSS
- ブラウザのデフォルトスタイルを一度全てリセットする考え方。
- Tailwind CSS も reset CSS の思想を含んでいる。

##### normalize.css
- ブラウザのデフォルトスタイルを活かしつつ、差異を整えるための CSS。
- https://necolas.github.io/normalize.css/
- 町田さんは以前から reset CSS を使用しており、Tailwind の普及によって「時代が追いついてきた」と感じているとのこと。

##### CSS の書き方・考え方
- `margin-inline: auto;`
  - 左右方向にのみ`margin`を設定できる。
  - ショートハンドという短く書かれた記法。
  - `start`は書字方向の開始側（左）、`end`は終了側（右）。（日本語の横書きでは、左 → 右）
- `margin-block`
  - 上下方向に`margin`を設定できる。
  - `start`が上、`end`が下。

##### 町田さんのコーディングの工夫・流儀
- クラス名の独自ルール
  - `l--`：layout の略。
  - レイアウト用スタイルであることを明示するためのネームスペース。
- 色指定の手癖
  - 一旦、`red`を指定して、カラーピッカーを表示するためのテクニック。

##### HTML / CSS設計について
- `h1`単体に装飾を当てるのではなく、`header`領域全体にデザインを適用する。
- `h1`はあくまで「Header領域内のテキスト」という認識を持つ。
- 見た目を整えるためには、HTML構造（`div`）の設計が重要。
  - `div`自体は意味を持たないが、レイアウト設計のための「棚」として使う。
  - どんな棚を作るか、という設計が重要。
- 昔は`div`が使えず、テーブルレイアウトで Webページを作っていた。
- 現在は`grid`や`flex`により設計の自由度が上がっているが、まだ進化の途中。

##### DevTools の便利機能
- `Edit HTML`：HTML をその場で書き換えられる。
- `Computed`：実際に適用されているスタイルを確認できる。
- `display: flex`
  - 子要素に影響し、横並びのレイアウトが可能。
- `gap`
  - 要素間の余白は`margin`ではなく`gap`を使うのが望ましい。

##### デザインの考え方
- 色のコントラストは`Contrast ratio`を確認する。
  - `AA`レベルを満たすことが目標。
- ホバー字のスタイル変更は、`:hover`。
- デザイン確認時は、あえて長い文字列を入れて確認する。
```shell
# 二行以上文字列を入れること！
Webデザインシステムの構築方法Webデザインシステムの構築方法Webデザインシステムの構築方法Webデザインシステムの構築方法Webデザインシステムの構築方法Webデザインシステムの構築方法Webデザインシステムの構築方法Webデザインシステムの構築方法
```
- Webデザインでは「4 の倍数」を意識する。
    - `rem`と相性が良い。
- `rem`：root em（ルート エム）。
- 見出しは行間を詰めた方が読みやすい。
  - 特に一行目と二行目の間隔が重要。
