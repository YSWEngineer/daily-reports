## 💡 本日の学び・気付き
### 1. wcコマンドとは？
#### 基本
`wc`= word count の略。テキストの「行数・単語数・バイト数」を数えるコマンド。本のページ数・行数・文字数を一気に数えてくれる**カウンター**のようなイメージ。
#### 出力の読み方
```shell
~/test_file % wc 1.txt
       3       9      45 1.txt
```
左から「行数 → 単語数 → バイト数 → ファイル名」の順番で表示される。この順番は固定。
#### 行数とは？
改行が何回あるかを数える。
```
1  Hello world
2  This is a test file
3  Good morning
4  （改行）
```
↑改行が3回なので、行数は3。
#### 単語数とは？
スペース・タブ・改行で区切られたまとまりの数。
```
Hello world  # 2単語
This is a test file  # 5単語
Good morning  # 2単語
# 計 9単語
```
#### バイト数とは？
ファイルの「データの重さ」のこと。半角英数字1文字 = 1バイトで数える。スペースや改行もそれぞれ1バイトとしてカウントされる。
※ 「文字数」と似ているが厳密には違う。日本語は1文字で3バイトになるため、日本語が混ざると文字数とバイト数がずれる。

### 2. オプション-l / -w / -c
#### オプションとは？
コマンドの後ろに`-l`のように付け加えることで、「表示する内容を絞り込む」指示。

|オプション   |意味   |表示されるもの   |
|---|---|---|
|なし   |全部表示   |行数・単語数・バイト数   |
|-l   |lines（行）   |行数だけ   |
|-w   |words（単語）   |単語数だけ   |
|-c   |bytes（バイト）   |バイト数だけ   |
#### 実際の動き
```shell
~/test_file % wc -l 1.txt
       3 1.txt  # ←行数だけ
~/test_file % wc -w 1.txt
       9 1.txt  # ←単語数だけ
~/test_file % wc -c 1.txt
      45 1.txt  # ←バイト数だけ
```
#### 組み合わせパターン
オプションはまとめて書ける。表示順は「行 → 単語 → バイト」で固定。
```shell
~/test_file % wc -lw 1.txt
       3       9 1.txt  # ←行数と単語数
~/test_file % wc -lwc 1.txt
       3       9      45 1.txt  # ←全部（オプションなしと同じ）
```

### 3. ファイルを複数指定したとき
#### 1ファイルのとき
```shell
~/test_file % wc 1.txt
       3       9      45 1.txt
```
#### 複数ファイルのとき
`total`が追加される。
```shell
~/test_file % wc 1.txt 2.txt 
       3       9      45 1.txt
       4       7      43 2.txt
       7      16      88 total  # ←全ファイルの合計が自動で追加
```
※ total行は複数ファイルの時だけ表示される。1ファイルのときは表示されない。

### 4. パイプ | とは？
#### パイプの仕組み
パイプ`|`は「左のコマンドの出力を、右のコマンドの入力に渡す」つなぎ役。
```shell
% ls -l | wc
```
水道管（パイプ）のイメージ。左から流れてきたデータを右に流し込む。ファイルを経由せず、直接データを渡せる。
#### 標準入力とは？
パイプで渡されたデータのことを「標準入力（`stdin`）」と呼ぶ。`wc`はファイル名の引数がないとき、自動的に標準入力からデータを受け取る。
- `stdin`：standard input の略
```shell
~/test_file % ls -l | wc
      27     236    1461  # ←ls -lの出力結果の行数・単語数・バイト数
```
ファイルから読むのではなく「`ls -l`が出力した文字列」をそのまま数えているため、ファイル名は表示されない。

### 5. ls -l | wc の組み合わせ
#### ls -l の出力を wc で数える
`ls -l→ ファイル一覧を出力` | `wc→ その行数などを集計`
```shell
# ls -lの出力
~/test_file % ls -l
total 40
-rw-r--r--@ 1 yoshiwo  staff    45 Mar 19 17:49 1.txt
（中略）
-rw-r--r--  1 yoshiwo  staff     0 Sep 13  2025 postcss.config.js

# wcがls -lの出力を数えた結果
27     236    1461
```
