## 何を作るのか？（目的）
・ Unixコマンド`wc`を Ruby で再現するスクリプト（`wc.rb`）を作る
・テキストの「行数・単語数・バイト数」を集計して表示する

## 入力パターン（2種類に対応する）
### パターン1：ファイル指定
→ 引数にファイル名を渡す方法。ファイルが一つでも複数でも動く。
```shell
% ruby wc.rb 1.txt
% ruby wc.rb 1.txt 2.txt
```
### パターン2：パイプ（標準入力）
→ パイプで前のコマンドの出力を受け取る方法。
```shell
% ls -l | ruby wc.rb
% ruby ls.rb | ruby wc.rb
```

## オプション（6パターンすべてに対応する）
### 対応が必要なオプション一覧
- [ ] オプションなし → 行数・単語数・バイト数をすべて表示
- [ ] `-l`のみ → 行数だけ表示
- [ ] `-w`のみ → 単語数だけ表示
- [ ] `-c`のみ → バイト数だけ表示
- [ ] `-lw` / `-wc` / `-lc` → 指定した二つを表示
- [ ] `-lwc` → 全部表示（オプションなしと同じ）

表示順は「行数 → 単語数 → バイト数」で固定（本物の`wc`と同じ挙動で）。
オプションの指定順は関係ない。 

## 複数ファイルのときの動き
### total行を追加する
- [ ] ファイルが一つだけのときは、`total`行は不要
- [ ] ファイルが二つ以上のときは、各ファイルの結果の下に`total`（合計）行を表示する
```shell
# ファイルが一つだけのとき
~/wc_practice % wc 1.txt      
       3       9      45 1.txt

# ファイルが二つ以上のとき
~/wc_practice % wc 1.txt 2.txt 
       3       9      45 1.txt
       4       7      43 2.txt
       7      16      88 total
```

→ ファイル名の引数なしでパイプから受け取ったデータを集計して表示できるようにする。

#### そのためには......
- 今のコードは`ARGV[0]`にファイル名がある前提で動いている。
- パイプで実行したとき（`ARGV`が空のとき）はどう対応すれば良いか、考えてみる🤔
```shell
# 考え方
・ ARGV[0]にファイル名がある → File.read で読む（Step2で完成済み）
・ ARGVが空 → $stdin から読む（Step3でやること）
```
→ `ARGV`が空かどうかを判断するメソッドは何だろう？
→ `empty?`メソッドは文字列や配列が空かどうかにも使えるメソッド

#### empty?メソッドについて
`empty?`メソッドは文字列だけでなく、配列が空かどうかにも使える。

|対象   |例   |結果   |
|---|---|---|
|文字列   |"".empty?   |true   |
|文字列   |"hello".empty?   |false   |
|配列   |[].empty?   |true   |
|配列   |["1.txt"].empty?   |false   |

#### 復習：述語メソッド
ちなみに、`?`で終わるメソッドは慣習として真偽値（`true`または`false`）を返すメソッドになっている。
このようなメソッドを**述語メソッド**という。

|メソッド   |意味   |
|---|---|
|ARGV.empty?   |ARGVが空かどうか？   |
|"hello".include?("he")   |文字列を含んでいるか？   |
|[1, 2, 3].include?(2)   |配列に含まれているか？   |

#### $stdinとは何か？
Ruby のプログラムがデータを受け取る方法は2種類ある。
```
# 方法1：ファイルから受け取る
% ruby wc.rb 1.txt
→ 1.txt の中身を File.read で読む

# 方法2：パイプから受け取る
% cat 1.txt | ruby wc.rb
→ cat の出力を $stdin で受け取る
```
```
# パイプのイメージ
cat 1.txt　→　パイプ（|）　→　ruby wc.rb
　↑                              ↑
データを流す側              $stdin で受け取る側
```
```
# 水道管のイメージ
cat 1.txt = 蛇口（水を出す）
パイプ | = 水道管（水を運ぶ）
$stdin = 蛇口（水を受け取る）
```

なぜ`$stdin`を使うのか？ → **パイプで流れてきたデータはファイルではない**ため`File.read`では読めない。パイプ経由のデータを受け取る専用の入り口が`$stdin`。

> 補足：パイプで流れるのは「ファイルそのもの」ではなく「ファイルの中身（文字列）」

```
# ファイルから読む
content = File.read('1.txt')

# パイプから読む
content = $stdin.read
```
`.read`を使う点は同じ。受け取る入り口が違うだけ。
具体的に表すと......
```shell
$ cat 1.txt | ruby wc.rb

① cat 1.txt が 1.txt の中身を読む
　→「Hello world
　　 This is a test file
　　 Good morning」

② その中身がパイプ（|）で流れる
　→ データとして wc.rb に渡される

③ wc.rb が $stdin でそのデータを受け取る
　→ content = $stdin.read
```

|   |ファイル指定   |パイプ   |
|---|---|---|
|渡されるモノ   |ファイルのパス(1.txt)   |ファイルの中身（文字列）   |
|読み方   |File.read(ARGV[0])   |$stdin.read   |
|結果   |同じ文字列がcontentに入る   |同じ文字列がcontentに入る   |

どちらの方法でも最終的に`content`に入る中身は同じ。受け取り方が違うだけ！

#### 分岐をコードに組み込む
「`content`に何を入れるか」だけが変わる。それ以降の処理（カウント・表示）は同じ。
```ruby
# ここだけ分岐する
if ARGV.empty?  # もしARGVが空なら
  content = $stdin.read  # パイプから読みこんでcontentに代入する
else  # そうでなければ
  content = File.read(ARGV[0])  # ARGV[0]のファイルを読み込んでcontentに代入する
end  # 分岐終了

# ここから先は同じ
lines = content.lines.count
words = content.split.size
bytes = content.bytesize

print lines.to_s.rjust(8)
print words.to_s.rjust(8)
print bytes.to_s.rjust(8)
print " #{ARGV[0]}"
print "\n"
```
#### まとめると
- `$stdin`：パイプで流れてきたデータを受け取る入り口
- `ARGV.empty?`：パイプかファイルかを判断する
- `content`：パイプ、ファイルどちらの方法でも同じ変数に格納する
