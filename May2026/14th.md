## 💡 本日の学び・気付き
### 【Phase 1】概念を整理
#### SSLとは？
普通の HTTP通信は「葉書」のようなもの。中身が丸見え。
SSL で暗号化された HTTPS通信は「封筒」のようなもの。中身が外から見えない。
SSL を使うには**証明書**が必要。証明書には2種類ある。
```
【Let's Encrypt証明書】     【自己証明書（オレオレ証明書）】
- 無料で取得できる            - 自分で作る
- ブラウザが信頼する          - ブラウザに「危険」と表示される
- 本番環境向き               - 学習・検証向き（仕組みを理解しやすい？）
```

#### 復習：VirsualHostとは？
1台のサーバー（1つのIPアドレス）で、ドメインごとに別サイトを提供する仕組み。
```
クライアント
    |
    | ysw-engineer.com へアクセス
    ↓
  nginx（1台）
    |--- site1.ysw-engineer.com → /var/www/site1/index.html
    |--- site2.ysw-engineer.com → /var/www/site2/index.html
```
nginx では`server { }`ブロックを複数書くことで実現する。

#### 証明書の構成ファイル
SSL には2つのファイルが必要。
|ファイル   |役割   |
|---|---|
|秘密鍵   |サーバーだけが持つ鍵。絶対に外に出さない   |
|証明書   |公開情報。ブラウザに渡して「このサーバーは本物です」と証明する   |

### 【Phase 5】nginx設定ファイルにHTTPS用サーバーブロックを追記
<details><summary> ⚠️ ネタバレに注意</summary>

#### site1・site2の設定ファイルを編集する
以下のコマンドを実行する。
```
sudo vim /etc/nginx/sites-available/site1
```
site1 の設定ファイルの中身を編集する。
```
server {
    listen 80;
    server_name site1.ysw-engineer.com;

    root /var/www/site1;
    index index.html;

    access_log /var/log/nginx/site1_access.log;
    error_log /var/log/nginx/site1_error.log;

    location / {
        try_files $uri $uri/ =404;
    }
}

# 以下の内容を新しく追加
server {
    listen 443 ssl;
    server_name site1.ysw-engineer.com;

    ssl_certificate     /etc/letsencrypt/live/site1.ysw-engineer.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/site1.ysw-engineer.com/privkey.pem;

    root /var/www/site1;
    index index.html;

    access_log /var/log/nginx/site1_access.log;
    error_log /var/log/nginx/site1_error.log;

    location / {
        try_files $uri $uri/ =404;
    }
}
```
