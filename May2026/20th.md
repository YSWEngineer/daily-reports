### 修正その1：HTTPとHTTPSで重複している設定をまとめる
今、`root`, `index`, `access_log`, `error_log`, `location`が HTTP用と HTTPS用の両方のブロックに同じ内容で書かれている。
これを解消するには共通設定を別ファイルに切り出す方法が一般的。
#### 修正その1：重複している設定をまとめる理由
現在の設定ファイルを見ると、HTTP用と HTTPS用のブロックでこれだけ同じ内容が繰り返されている。
```
# site1の設定ファイルで重複している箇所
root /var/www/site1;
index index.html;
access_log /var/log/nginx/site1_access.log;
error_log /var/log/nginx/site1_error.log;
location / { try_files $uri $uri/ =404; }
```
```
# site2の設定ファイルで重複している箇所
root /var/www/site2;
index index.html;
access_log /var/log/nginx/site2_access.log;
error_log /var/log/nginx/site2_error.log;
location / { try_files $uri $uri/ =404; }
```
同じ内容が HTTP と HTTPS の2箇所に書いてあると、将来変更が必要になったとき2箇所を同時に直さないといけない。
1箇所直し忘れると HTTP と HTTPS で動作が変わってしまうバグの原因になる。
プログラミングの世界では同じことを繰り返し書かない**DRY原則**（**D**on't **R**epeat **Y**ourself）に則って書き直す。

#### 修正
共通の設定を別ファイルに切り出して、HTTP用と HTTPS用のブロックから`include`で読み込む。
```
変更前
server { listen 80;    ... 共通設定 ... }
server { listen 443 ssl; ... 共通設定 ... }

変更後
server { listen 80;      include 共通設定ファイル; }
server { listen 443 ssl; include 共通設定ファイル; }
```
