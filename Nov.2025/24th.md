## 💡 本日の学び・気付き
### Cookie について
- Cookie は正確には`HTTP COOKIES`という。
- Stateless Protocol（ステートレスプロトコル）：送られてきたデータリンクが Webページが配信されると直ちに閉じられるプロトコル（通信手順）のこと。
- HTTP Cookie は、Webサーバからユーザのコンピュータに送られる情報の一部で、データ伝達の`HTTP Header`と呼ばれる物の中で送信され、Webブラウザ中に内部的に保存される。そして、次にオコの同じサーバにアクセスしたときに、サーバは HTTP Cookie を読み戻しユーザの認識に利用する。
- HTTP Cookie は最大 4KB。
- HTTP Cookie に保存されている情報
  - NAME=VALUE
  - expires=DATE
  - domain=DOMAIN_NAME
  - path=PATH
  - secure
- HTTP Cookie には Persistent Cookie（パーシステントクッキー）と Session Cookie（セッションクッキー）の2通りがある。
- Session Cookie は一時的な Cookie で、インターネットメールプロバイダにログインするときなどに使用される。最初のログインでメールを表示させようとするたびにアクセス情報（ID やパスワード）を入力しなくてよいのは、この Session Cookie のおかげ。また、オンラインバンクの認証も Session Cookie が使用されている。
- Seesion Cookie はサーバに対するセッションが終了した時点（ブラウザを終了した時点）で破棄される。
- Persistent Cookie はフォーラム等に利用されている。一回ログインすると一定期間の間にそのフォーラムにアクセスすると自動ログインされるのは、Persistent Cookie はサーバに読み戻されていることによるもの。Session Cookie とは異なり、Webブラウザを終了しても Webブラウザ内部に保存されている。Persistent Cookie を削除したい場合は、Webブラウザの設定で「終了時に Cookie を削除する」にチェックを入れる。
- そもそも Cookie の目的は、ステートレス（状態を維持できない）プロトコルである HTTP 上で、状態管理ができるようにすること。
- Cookie の仕組み
  - ①サーバが`Set-Cookie`というフィールドを HTTPヘッダに載せてレスポンスを返す。
  - ②レスポンスを受け取ったクライアントは Cookie をブラウザに保存する。そして、それ以降のリクエストには受け取った Cookie をそのままリクエストするようになる。

### セッションとは？
- セッション（Session）：一連のインタラクティブな操作のこと。言い換えると「サーバ側に保存される一時的なユーザ情報の入れ物」。
  - インタラクティブ：情報の送り手と受け手が相互に情報をやり取りできる状態。
- Web は本来、毎回のアクセスが独立したリクエストで、いちいち覚えていない。たとえば、「さっきログインしたのに、次のページに行ったらログアウトしたことになっている」
→ Web はもともと覚える仕組みがないから、これが普通。これを解決するために登場したのがセッション。
- セッションがしてくれること
  - ログイン状態
  - ユーザID
  - （ECサイトなら）カートの中身
  - 一時的な入力内容
  - ページ間で引き継ぎたい情報

### HTTP Cookie とセッションの違いについて
##### HTTP Cookie
- **ブラウザ側に保存するデータ**

##### セッション
- **サーバ側に保存するデータ**
