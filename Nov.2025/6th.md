## 💡 本日の学び・気付き
### キーワード引数
**キーワード引数**を使うと、**引数名と値のペア**で引数を渡せるようになる。
```ruby
# キーワード引数を使う場合のメソッド定義の構文
def メソッド名(引数1: 引数1の値, 引数2: 引数2の値, ...)
  実行したい処理
end
```

### キーワード引数と通常の引数を組み合わせる
キーワード引数は通常の引数と組み合わせて用いることができる。
```ruby
def func(a, b: 1, c: 2)
  ⋮
end
```
↑このように定義した場合、`a`は必須のパラメータ、`b`と`c`はキーワード引数となる。このメソッドを呼び出すときは、最初の引数に続けて**キーワード形式の引数を指定**する。
```ruby
func(1, b: 2, c: 3)
```

### ハッシュで引数を渡す
ハッシュをキーワード引数として渡すことができる。キーはシンボルでなければならない。
```ruby
def area(x: 0, y: 0, z: 0)
  xy = x * y
  yz = y * z
  zx = z * x
  (xy + yz + zx) * 2
end

args1 = {x: 2, y: 3, z: 4}
p area(args1)  #=> 52

args2 = {x: 2, z: 3}  # yを省略する
p area(args2)  #=> 12
```

### 配列を引数に展開する
メソッドに引数を渡す場合に、配列を展開してメソッドの引数にすることもできる。メソッドの呼び出しに「`*配列`」の形式で引数を指定すると、配列そのものではなく、配列の要素が先頭から順にメソッドの引数として渡される。ただし、配列の要素の数とメソッドの引数の数は一致していなければならない。
```ruby
def foo(a, b, c)
  a + b + c
end

p foo(1, 2, 3)     #=> 6

args1 = [2, 3]
p foo(1, *args1)   #=> 6

args2 = [1, 2, 3]
p foo(*args2)      #=> 6
```
- `foo`というメソッドは**3つの引数**（`a`, `b`, `c`）を受け取る。
- `p foo(1, 2, 3)`のとき、
  - `a = 1`
  - `b = 2`
  - `c = 3`となり、`1 + 2 + 3`の結果である`6`が出力される。
- `args1`は`[2, 3]`という**配列**。
- `*args1`と書くと、Ruby はこの配列をばらばらに展開する。

つまり、
```ruby
p foo(1, *args1)
```
は実際には以下のように扱われる👇🏻
```ruby
p foo(1, 2, 3)
```
よって結果は同じく`6`になる。
```ruby
args2 = [1, 2, 3]
p foo(*args2)
```
これも、`foo(1, 2, 3)`と同じ意味。配列`[1, 2, 3]`の要素が順番に引数`a`, `b`, `c`に代入される。→結果は`1 + 2 + 3 = 6`。

### 引数にハッシュを渡す
ハッシュオブジェクトは通常「`{ ~ }`」という形式で書くが、メソッドの引数にハッシュを渡す場合は、「`{ }`」を省略できる。
```ruby
def foo(arg)
  arg
end

p foo({"a"=>1, "b"=>2})  #=> {"a"=>1, "b"=>2}
p foo("a"=>1, "b"=>2)    #=> {"a"=>1, "b"=>2}
p foo(a: 1, b: 2)        #=> {:a=>1, :b=>2}
```
3つの書き方の比較
|書き方   |実際に渡される値（ハッシュ）   |キーの型   |
|---|---|---|
|foo({"a"=>1, "b"=>2})   |{"a"=>1, "b"=>2}   |文字列   |
|foo("a"=>1, "b"=>2)   |{"a"=>1, "b"=>2}   |文字列   |
|foo(a: 1, b: 2)   |{:a=>1, :b=>2}   |シンボル   |

### 読みやすいプログラムを書くこと
プログラムの見た目を良くするためのポイントとして、次の 3点が挙げられる。
- **改行と「;」セミコロン**
- **インデント**
- **空白**
#### 改行と「;」
改行は文の区切りに使えることが挙げられる。改行以外に、文の区切りになる記号として`;`がある。1行に複数の文を書きたい場合に使える。
```ruby
str = "hello"; print str
```
と書くのは、
```ruby
str = "hello"
print str
```
と書くのと同じ意味になる。
この文法は、改行そのものを自然な区切りとみなしたほうが、プログラムを書きやすく読みやすいという考えを表している。1行にいくつもの操作を書くよりも、適切に改行することが、読みやすいプログラムへの第一歩となる。
#### インデント
プログラムの行頭に空白文字をいくつか並べて、まとまりを強調するために使う。
```ruby
if a == 1
  print message1
  print message2
end
```
#### 空白
空白の長さは揃えて、バランスよくすることに気を付ける。
```ruby
a+b          # ○好ましい書き方
a + b        # ○好ましい書き方
a +b         # △好ましくない書き方
a+ b         # △好ましくない書き方
```
#### よいスタイル
よいスタイルを身に付けるには、他の人が書いた Ruby のプログラムを読んでそれを真似るところから始めるのがよい。
